欢乐小玩家系统
系统级 PRD + SOP（整合版）
一、系统概述（PRD）
1.1 系统定位

欢乐小玩家系统是一套 “线下门店 + 微信生态 + 商城 + 供应链” 的一体化业务系统，核心目标是：

通过 门店 + 店员 + 企业微信 完成用户拉新

通过 小程序商城 完成交易闭环

通过 平台运营后台 管控商品、活动、订单、售后

通过 供应商协作 完成履约与售后处理

1.2 系统核心能力地图
用户增长 → 商品管理 → 下单支付 → 履约物流 → 售后退款


对应模块：

用户与会员体系

商品与供应商管理

商城交易与支付

订单与物流履约

售后与退款

营销活动与消息推送

二、系统角色与职责模型

三、核心业务流程拆解（PRD + SOP）
3.1 用户拉新与注册流程
PRD 说明

以 门店 + 店员 为拉新入口

以 企业微信 为用户关系承载

以 商城注册 为交易入口

SOP 流程

用户到店消费

店员引导用户扫码关注企业微信

企业微信向用户推送：

新用户优惠

商城注册链接

用户点击跳转商城完成注册

用户成为欢乐小玩家会员

3.2 商品上架与供应商管理（你之前那张图）
PRD 说明

商品不支持供应商自助上架

全流程由平台运营管控

SOP（精简版）

运营与供应商洽谈商品

运营在后台配置：

商品信息

图片标签

售价 / 协议价

关联供应商

商品上架

商城自动展示

3.3 活动配置 & 消息推送流程
PRD 说明

活动统一在后台配置

消息通过 企业微信 触达用户

海报可跳转商城商品 / 活动页

SOP

平台运营在后台配置活动

新人优惠

折扣 / 海报

后台生成活动 / 新品海报

企业微信自动或手动推送海报

用户点击海报跳转商城

用户浏览 / 下单

3.4 用户选购 & 支付流程
PRD 说明

商城为唯一交易入口

支付走微信支付

SOP

用户浏览商品

选择商品下单

商城调用微信支付

支付成功生成订单

支付 UID 写入订单

3.5 订单流转 & 发货流程
PRD 说明

订单由平台运营驱动履约

当前为 人工对接供应商发货模式

SOP

支付完成 → 订单生成

后台通知平台运营

运营联系供应商发货

供应商发货并提供快递单号

运营在后台填写快递单号

订单状态更新

用户查看物流信息

3.6 物流与订单状态
PRD 说明

物流信息非实时自动对接

查询物流可能产生费用

SOP

后台记录物流状态

用户在商城查看订单状态

系统根据需要查询物流接口

3.7 售后 & 退款流程（完整闭环）
PRD 说明

企业微信作为客服入口

商城作为售后单承载系统

退款通过微信支付原路返回

SOP

用户联系企业微信咨询售后

运营引导用户在商城提交售后申请

后台生成售后单

平台运营审核售后

联系供应商确认退货

用户寄回商品（填写快递单号）

供应商确认收货

运营确认退款

微信支付自动退款

售后完成

四、系统状态机（核心）
4.1 商品状态

草稿 → 已上架 → 已下架

4.2 订单状态

待支付 → 已支付 → 已发货 → 已完成 → 售后中 → 已退款
