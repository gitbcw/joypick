企业微信 × 小程序联动说明文档（精简可执行版）
1. 两者能否互通？

能互通。企业微信可与微信用户聊天，小程序可在企业微信里运行。

企业微信员工可直接给微信用户发送小程序卡片（1v1）。

2. 可实现的联动方式

小程序内放企业微信客服入口
用户点客服 → 跳企微私聊。

企业微信发送小程序卡片
员工可直接把“小程序页面（带参数）”发给微信用户。

行为 → 自动触发消息（需开发）
小程序埋点 → 事件同步 CRM → 在 48 小时窗口内自动推送消息。

扫码绑定关系（关键能力）
通过企业微信“活码”实现：
扫码 = 加企微 + 进入小程序（带来源参数）

小程序在企业微信工作台运行
可做内部管理工具（订单、售后等）。

3. 企业微信 → 微信用户发送消息规则

人工 1v1：可主动发消息给微信用户。

批量群发：不允许对微信用户群发，只能单聊。

自动消息：需用户产生动作（加好友、发消息、点击活动），才触发自动回复。

4. 自动触达逻辑

用户在小程序有行为（下单、领券等）
→ 行为同步到企微 CRM
→ 开启 48 小时会话窗口
→ 系统可自动推送营销/提醒消息（瑞幸同款玩法）

关键：小程序动作会被视为“有效互动”，从而合法推送。

5. 扫码绑定关系（最关键能力）

一码两用：加企微 + 进小程序 + 带来源参数

流程：

用户扫 企业微信活码

加企微员工

企业微信自动发“欢迎语 + 小程序卡片”

用户进入小程序，系统知道：

来自哪个员工/渠道

通过哪个活动进入

用于私域运营、自动化分配、行为追踪。

可实现：精准营销、员工业绩归因、活动渠道统计。

6. 开发复杂度说明

基础玩法（客服入口、发送小程序卡片）：很简单

行为埋点 + CRM 自动化：中等复杂度

完整打通（瑞幸级别）：需要后端 + SCRM 配合

7. 你们项目的建议做法

先做基础版可落地功能：

小程序客服入口 → 企微

企微欢迎语自动发活动小程序卡片（零成本）

再做高级版（可单独收费）：

用户行为埋点

与 SCRM 对接 → 自动化消息

把“企业微信配置步骤”做成固定 SOP
减少每次重复劳动。

关键点总结：企业微信负责“触达与私域”，小程序负责“页面与交易”；两者靠“扫码绑定 + 小程序卡片 + 行为事件”实现深度联动。
